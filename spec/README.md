# First Layer

This folder is the first architectural layer of the benchmark.

It defines:

- what the benchmark is;
- what it compares;
- what is fixed across all runs;
- how the repository is organized;
- how human judgment is recorded.

Files:

- `CHARTER.md` — scope and benchmark purpose.
- `RULES.md` — non-negotiable comparison rules.
- `REPO_LAYOUT.md` — folder responsibilities.
- `HUMAN_REVIEW.md` — how manual scoring works.
- `TECHNICAL_PROFILE.md` — required capability check before content tasks.
