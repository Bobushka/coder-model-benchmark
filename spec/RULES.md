# Rules

## Fixed across all compared harnesses

- same `AGENTS.md`;
- same task text;
- same starting files;
- same time limit;
- same submission contract;
- same reviewer.

## Not fixed across harnesses

- native harness UX;
- native tool access model;
- native planning style;
- native transcript shape.

These are part of what is being evaluated.

## Comparison rules

- no manual help during a run;
- no post-hoc cleanup before review;
- no changing the task wording per harness;
- no hidden second attempt unless the run mode explicitly allows it.
