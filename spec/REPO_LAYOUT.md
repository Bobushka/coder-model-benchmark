# Repository Layout

- `spec/` — benchmark policy and source-of-truth docs.
- `task-packs/` — normalized tasks.
- `runner/` — adapter contract and execution machinery.
- `results/` — raw runs and human review artifacts.
- `models/` — model catalog and adapter mapping.

The repo must stay benchmark-only.
Do not mix unrelated experiments, personal notes, or external project code here.
